## 零、严格 5 分钟压缩口播版

这一版适合正式上台直接讲，核心是“任务是什么、模型怎么做、实验说明什么、结论是什么”。第 5、7、8 页如果时间不够，只点一句结论。

### 第 1 页，约 20 秒

各位老师好，我的毕业论文题目是《基于 KAN 超图的药物协同预测研究》。本文研究的是抗癌联合用药中的协同预测问题：输入两个药物的分子结构和一个细胞系的表达特征，模型输出它们是否协同，或者预测连续的 Loewe 协同分数。我的重点不是简单证明 KAN 一定比 MLP 好，而是研究 KAN 放在药物图编码、预测头、超图交互这些不同位置时，哪一种结构更合理。

### 第 2 页，约 35 秒

这个任务有三个主要难点。第一，药物组合空间很大，药物数量一多，二药组合数量会快速增长。第二，协同正例比例低，本文数据里正例大约 14.8%，所以 AUPR 比 accuracy 更重要。第三，random split 可能让训练集和测试集共享药物，容易高估泛化能力，所以我重点关注 cold-drug，也就是测试药物在训练中没有出现的情况。本文主要贡献是：构建了 16 万级样本，比较了 KAN head、DrugKAN、三元 KAN 超图和药物对聚合图，并补充了扰动、响应曲线和原子 saliency 解释。

### 第 3 页，约 35 秒

数据方面，一个样本由药物 A 分子图、药物 B 分子图、细胞系表达向量、分类标签和 Loewe 分数组成。药物由 SMILES 经过 RDKit 转成分子图，节点是原子，边是化学键；主实验细胞系表达向量维度是 946。最终保留 160,228 条有效样本，包含约 1,896 个药物和 156 个细胞系。分类任务报告 AUPR 和 AUC，回归任务报告 RMSE 和 MAE。cold-drug 划分中我做了泄漏检查，训练集和测试集药物重叠数是 0。

### 第 4 页，约 55 秒

模型整体分三部分：drug 端、cell 端和交互模块。drug 端把两个药物分别看成分子图，通过 GCN/GAT 或 KAGNN 编码成 128 维 drug embedding。如果使用 DrugKAN，就是把图卷积中的线性变换换成 KANLinear，让 KAN 在原子邻域消息传递里学习非线性函数。cell 端把 946 维表达向量经过全连接网络编码，再对齐到 128 维。

后面的交互模块是本文比较的核心。MLP baseline 直接拼接 drug A、drug B 和 cell 三个向量。KAN head 是把最后预测头替换为 KAN。HgKAN-HG 把 drug A、drug B 和 cell 作为三个节点，用一条超边连接。HgKAN-Agg 则先把两个药物合成 drug-pair 节点，再让 drug-pair 和 cell 交互。最终结果说明，药物对聚合图比直接三元超图更稳定。

### 第 5 页，约 25 秒

random split 主要检验已见药物空间内的插值能力。这里 MLP baseline 很强，AUPR 为 0.5889，AUC 为 0.8833；直接 KAN head 下降到 AUPR 0.5064，AUC 0.8561。这说明 KAN 不能简单无结构地替代最终 MLP，尤其当输入是拼接后的高维隐藏向量时，每一维缺少明确药理含义，KAN 的优势不一定能发挥出来。

### 第 6 页，约 50 秒

cold-drug 是本文最重要的实验，因为测试药物没有在训练中出现，更接近新药组合筛选。HgKAN-Agg 的 AUPR 为 0.2765，略高于 MLP baseline 的 0.2758；AUC 从 0.7333 提升到 0.7452。回归上，RMSE 从 18.3997 降到 17.6763，MAE 从 13.6750 降到 12.8615。这里 AUPR 提升很小，所以不能夸大为显著领先，但四个指标方向一致，说明药物对聚合图在当前设置下是最明确的正结果。

### 第 7 页，约 25 秒

随机种子实验用于检查稳定性。random split 下两个 seed 比较接近，但 cold-drug 波动明显，说明不同测试药物集合难度差异很大。因此本文结论要克制：当前结果支持 HgKAN-Agg 这个结构方向更有潜力，但后续仍需要更多 seed 和外部数据集验证。

### 第 8 页，约 25 秒

基函数和参数量实验说明两点。第一，B-spline KAN 理论上适合有明确连续含义的变量，比如剂量，但本文 KAN 输入主要是隐藏通道，所以 Fourier KAN 更稳定。第二，KAN 没有带来明显减参，模型参数主要来自细胞表达编码器和预测网络。因此本文贡献不在参数效率，而在结构化消息函数和解释性。

### 第 9 页，约 40 秒

可解释性部分选了一个 cold-drug 正样本：Lenalidomide 和 mitomycin C 在 IGROV1 细胞系上组合。原始协同概率是 0.9106；屏蔽 Lenalidomide 后降到 0.5997，屏蔽 mitomycin C 后降到 0.4152，同时屏蔽两药后降到 0.3711，说明模型确实依赖药物输入。药物对表示恢复到 15% 时，概率已经达到 0.6506，说明输出对药物对信息很敏感。原子 saliency 也落在两个药物的关键结构区域，但这只能作为模型解释证据，不能直接等同于生物机制证明。

### 第 10 页，约 30 秒

最后总结，本文的核心结论不是“KAN 全面优于 MLP”，而是 KAN 的效果高度依赖放置位置。直接 KAN head 效果较差，DrugKAN 在回归中有一定收益但分类不稳定，而 HgKAN-Agg 在 cold-drug 分类和回归主指标上方向一致优于 baseline。可以概括为一句话：结构先行，KAN 局部函数化。也就是先用图或超图结构明确药物、药物对和细胞背景的关系，再让 KAN 在局部消息函数中学习非线性。我的汇报结束，谢谢各位老师。

## 一、逐页讲稿

### 第 1 页：封面

各位老师好，我的毕业论文题目是《基于 KAN 超图的药物协同预测研究》。

本文研究的是抗癌联合用药场景中的药物协同预测问题。简单来说，给定两个药物的分子结构和一个细胞系的表达特征，模型需要判断这两个药物在该细胞系上是否可能产生协同作用，或者预测连续的 Loewe 协同分数。

我这篇论文的关键词有三个：药物协同、结构化交互和 KAN。最终想回答的问题不是“KAN 这个模块是不是一定比 MLP 强”，而是“在药物协同预测任务中，KAN 应该放在哪里才更合理”。

### 第 2 页：研究问题与贡献

药物协同预测不是普通的单分子性质预测。普通分子性质预测主要判断一个药物本身的活性或毒性，而药物协同预测的对象是一个三元关系：药物 A、药物 B 和细胞系背景。

这个任务有几个难点。第一，药物组合空间增长很快，如果有 n 个药物，仅二药组合就有 n 乘 n 减 1 再除以 2 种。第二，协同正例比例比较低，本文数据中正例比例约为 14.8%，所以不能只看 accuracy，更要看 AUPR。第三，random split 中训练集和测试集可能共享药物实体，容易高估泛化能力，所以我重点看 cold-drug，也就是测试药物没有在训练集中出现的场景。第四，药物发现任务需要一定可解释性，不能只给一个分数。

本文的贡献主要有四点。第一，构建了约 16 万条 DrugComb 相关药物组合样本。第二，系统比较了 KAN 的不同放置方式，包括 KAN head、DrugKAN、三元超图和药物对聚合图。第三，在 cold-drug 场景中验证了 HgKAN-Agg 的趋势优势。第四，构建了药物扰动、药物对响应曲线、KAN 函数曲线和原子 saliency 的解释流程。

### 第 3 页：数据与评价协议

本文每个样本可以写成一个五元组：药物 A 的分子图、药物 B 的分子图、细胞系表达向量、二分类协同标签和连续 Loewe 分数。

药物结构来自 SMILES，经过 RDKit 解析成分子图。图中的节点表示原子，边表示化学键。细胞系输入是基因表达向量，主实验实际维度是 946。最后保留成功解析分子结构、并且具有有效细胞表达和协同标签的样本，得到 160,228 条样本，约 1,896 个唯一药物和 156 个细胞系。

评价任务有两个。分类任务预测协同或非协同，主要报告 AUPR 和 AUC。回归任务预测 Loewe score，主要报告 RMSE 和 MAE。数据划分也有两类：random split 用于检验已有药物空间内的插值能力；cold-drug split 要求测试药物不出现在训练集中，更接近新药候选组合筛选。本文在 cold-drug 中做了泄漏检查，train-test 药物重叠数为 0。

### 第 4 页：模型设计

这一页是本文最核心的方法设计。整体模型分为三部分：drug 端药物图编码器、cell 端表达编码器，以及 drug-drug-cell 交互模块。

drug 端对两个药物分别编码。药物 A 和药物 B 都是分子图，先通过图神经网络得到图级 embedding。普通 GCN 或 GAT 可以作为 baseline；如果打开 `use_drug_kan`，图卷积中的线性变换会被 KANLinear 替换，这就是 DrugKAN。

cell 端是一个全连接表达编码器，把 946 维基因表达映射到隐藏表示。之后用 `EmbeddingAligner` 把 drug embedding 和 cell embedding 统一到 128 维。

交互模块是本文比较的重点。最简单的是 MLP baseline，也就是把药物 A、药物 B 和细胞表示拼接起来预测。KAN head 是直接把最后的 MLP 预测头换成 KAN。HgKAN-HG 是三元超图，把 drug A、drug B 和 cell 作为三个节点，用一条超边连接。HgKAN-Agg 是药物对聚合图，先把两个药物合成 drug-pair 节点，再让 drug-pair 和 cell 交互。本文最终主结果是 HgKAN-Agg。

### 第 5 页：Random split 结果

这一页是 random split 的结果。random split 中，测试样本的药物可能在训练集中出现过，所以它主要反映已见药物空间内的插值能力。

可以看到，MLP baseline 在 random split 中仍然很强，分类 AUPR 是 0.5889，AUC 是 0.8833。直接 KAN head 反而下降，AUPR 从 0.5889 降到 0.5064，回归 RMSE 也从 18.9204 增加到 19.4118。

这说明一个重要负结果：KAN 的理论表达能力不能自动转化为药物协同预测性能。尤其是当输入是药物 A、药物 B 和细胞表示拼接后的高维隐藏向量时，每一维并没有明确药理含义，KAN 的单变量函数未必能学到稳定规律。

所以 random split 的结论是：KAN 不能无结构地替代最终 MLP；是否有效取决于它所在的结构位置。

### 第 6 页：Cold-drug 主结果

这一页是本文最重要的实验。cold-drug 要求测试药物没有在训练中出现，因此更接近新药组合筛选场景。

从分类看，HgKAN-Agg 的 AUPR 为 0.2765，略高于 MLP baseline 的 0.2758；AUC 从 0.7333 提升到 0.7452。AUPR 的提升很小，所以这里不能夸大为显著领先，但 AUC 的方向是改善的。

从回归看，HgKAN-Agg 的 RMSE 为 17.6763，优于 MLP baseline 的 18.3997；MAE 为 12.8615，也优于 baseline 的 13.6750。也就是说，在分类和回归四个主指标上，HgKAN-Agg 的方向是一致优于 baseline 的。

同时可以看到，KAN head 在 cold-drug 中明显退化，AUC 只有 0.5590；DrugKAN 在回归中有一定收益，但分类不稳定；DrugKAN+Agg 叠加更多 KAN 后并没有继续变好。这个结果支持本文的核心观点：不是 KAN 越多越好，而是药物对聚合图给 KAN 提供了更合适的结构载体。

### 第 7 页：二分类随机种子稳定性

这一页是为了补充实验完整性，也避免过度包装结论。

可以看到，在 random split 中，MLP baseline 和 DrugKAN 两个 seed 的结果比较接近，说明插值性能相对稳定。但在 cold-drug 中，绝对指标随药物划分变化很明显。比如 MLP baseline 在 seed42 的 AUPR/AUC 是 0.2758/0.7333，而 seed43 下降到 0.0839/0.4646，说明 seed43 的测试药物集合更困难。

不过，在两个 cold-drug seed 中，DrugKAN+KAN-Agg 都优于 DrugKAN+KAN-HG，说明药物对聚合图方向比三元 KAN 超图更稳定。

因此这一页的结论比较克制：当前结果支持“药物对聚合图方向更有潜力”，但还不足以声称 cold-drug 分类已经统计显著领先。后续需要更多 seed 和外部数据验证。

### 第 8 页：KAN 基函数与参数量分析

这一页回答两个补充问题：第一，为什么主实验使用 Fourier KAN；第二，KAN 有没有减少参数量。

本文实现了 Fourier KAN 和 B-spline KAN。B-spline 有局部支撑和平滑性，理论上更适合显式剂量变量或剂量-反应曲线。但本文中的 KAN 输入主要是隐藏通道，不是真实剂量，所以 B-spline 的优势没有充分发挥。

实验上，B-spline DrugKAN 在 random split 下接近 baseline，但在 cold-drug 下 RMSE 为 19.0523，弱于 Fourier DrugKAN 的 17.8930，也弱于 HgKAN-Agg 的 17.6763。因此主实验采用 Fourier KAN。

参数量方面，所有模型总参数量都在四千万级，主要来自细胞表达编码器和预测网络。加入 KAN 后，KAN 参数大概从 0.9M 到 3M 多不等，并没有带来明显减参。因此本文不把参数效率作为主要贡献，而是强调结构化消息函数和解释性。

### 第 9 页：可解释性与案例分析

这一页展示一个 cold-drug 正样本案例：Lenalidomide 和 mitomycin C 在 IGROV1 细胞系上的组合。两种药物都没有出现在训练集中，所以这个案例可以检验模型是否在未见药物上利用了结构信息。

左边是药物扰动实验。原始协同概率是 0.9106；屏蔽 Lenalidomide 后下降到 0.5997，屏蔽 mitomycin C 后下降到 0.4152，同时屏蔽两药后下降到 0.3711。这说明模型预测不是只依赖类别先验或细胞背景，而是确实依赖药物输入。

中间是药物对表示保留响应曲线。把药物对表示从 0 逐渐恢复到原始表示时，预测概率也从非协同区间逐步上升。当恢复到 15% 时，概率已经达到 0.6506，越过 0.5 阈值。这说明模型输出对药物对信息有非线性敏感性。

右边是 KAN 函数曲线。这里要强调，KAN 曲线不是药物剂量-反应曲线，它的横轴是隐藏通道激活值；它只能说明模型内部消息函数学习到了非线性关系，不能直接解释为真实生物机制。

在论文中还进一步做了原子 saliency，发现 Lenalidomide 的高贡献原子覆盖 glutarimide 环核心，mitomycin C 的高贡献原子覆盖 quinone/mitosene 骨架。这些区域与已知药理相关结构有合理重合，但仍然是模型解释证据，不是机制发现的最终证明。

### 第 10 页：结论

最后总结一下。本文的最终结论不是“KAN 全面优于 MLP”，而是：在药物协同预测任务中，KAN 的效果高度依赖放置位置。

直接 KAN head 替换最终 MLP 效果较差，说明高维拼接表示不适合无结构 KAN 拟合。DrugKAN 在 cold-drug 回归中有一定收益，但分类不稳定，说明单药分子图编码只能解决部分问题。HgKAN-Agg 在 cold-drug 分类和回归主指标上方向一致优于 MLP baseline，是本文当前实验中最明确的正结果。

因此我把本文的方法经验概括为一句话：结构先行，KAN 局部函数化。图或超图结构负责提供药物拓扑、药物对和细胞背景的局部化归纳偏置；KAN 作为消息函数学习可视化的非线性关系。

我的汇报结束，谢谢各位老师，欢迎批评指正。

## 二、技术细节讲解：drug 端、超图端和数据流程

这一部分用于答辩问答，或者你想在方法页讲得更细时使用。

### 1. 整体输入和输出是什么？

一个样本的输入包括三类：

1. 药物 A 的分子图 `G_a`：节点是原子，边是化学键；
2. 药物 B 的分子图 `G_b`：同样是原子-化学键图；
3. 细胞系表达向量 `x_c`：主实验实际输入维度是 946。

样本还带有：

1. 分类标签 `y`：0 表示非协同，1 表示协同；
2. 连续 Loewe score：用于回归任务；
3. `drug_a_id`、`drug_b_id`、`cell_id`：用于在 batch 内构造超图或聚合图。

模型输出分两种：

1. 分类任务：输出 `[batch_size, 2]` 的 log-softmax，表示非协同/协同两个类别的对数概率；
2. 回归任务：输出 `[batch_size]` 的连续分数，训练时对 Loewe 分数做标准化，评价时还原到原始尺度。

### 2. batch 里数据长什么样？

PyTorch Geometric 会把一个 batch 里的多个分子图拼成一个大的不连通图。

对于药物 A：

1. `batch.x_a` 是所有药物 A 图的原子特征拼接；
2. `batch.edge_index_a` 是拼接后的边索引；
3. `batch.edge_attr_a` 是边特征；
4. `batch.x_a_batch` 表示每个原子属于 batch 中哪个样本。

药物 B 同理，有 `x_b`、`edge_index_b`、`edge_attr_b`、`x_b_batch`。

细胞系表达是 `batch.expression`，形状可以理解为 `[batch_size, 946]`。  
标签是 `batch.y`。  
用于建图的实体 ID 是 `batch.drug_a_id`、`batch.drug_b_id` 和 `batch.cell_id`。

### 3. drug 端具体怎么做？

drug 端对应代码里的 `KAGNN` 和 `KAGNN_node`。

第一步，原子特征编码。  
原始 `x` 是离散原子特征，先经过 OGB 风格的 `AtomEncoder`，变成连续节点表示，形状大致是：

```text
[所有原子数, 原子特征] -> [所有原子数, emb_dim]
```

本文主实验里 drug embedding 维度一般是 128。

第二步，做多层图消息传递。  
普通 GCN/GAT/GIN 是 baseline；如果开启 DrugKAN，则在图卷积内部使用 KANLinear。

以代码中的 `KAGCNConv` 为例，每层做的是：

```text
neighbor_mean_i = mean(h_j, j in N(i))
combined_i = concat(h_i, neighbor_mean_i)
transformed_i = KANLinear(combined_i) 或 Linear(combined_i)
h_i' = skip(h_i) + transformed_i
```

也就是说，KAN 不是直接对整个药物对做预测，而是在分子图的局部原子邻域中学习非线性节点更新函数。这样输入更有结构约束：每次只看当前原子和邻居原子聚合结果。

第三步，图级池化。  
经过 5 层消息传递后，所有原子节点表示通过 `global_mean_pool` 等方式聚合成一个 drug embedding：

```text
药物A图 -> h_a_raw: [batch_size, 128]
药物B图 -> h_b_raw: [batch_size, 128]
```

如果使用普通 GCN，这就是普通 drug encoder；如果使用 KAN 版本，就叫 DrugKAN。

### 4. cell 端具体怎么做？

cell 端是 `ExpressionNN`。输入是细胞系基因表达向量：

```text
expression: [batch_size, 946]
```

经过两层较大的全连接网络：

```text
946 -> 8192 -> 4096 -> 256
```

得到细胞系表示：

```text
h_e_raw: [batch_size, 256]
```

然后进入 `EmbeddingAligner`，把 drug 和 cell 对齐到统一维度 128：

```text
h_a_raw: [B, 128] -> h_a_aligned: [B, 128]
h_b_raw: [B, 128] -> h_b_aligned: [B, 128]
h_e_raw: [B, 256] -> h_e_aligned: [B, 128]
```

后面的交互模块都基于这三个 128 维向量。

### 5. MLP baseline 和 KAN head 怎么走？

MLP baseline 不构造额外图结构。它直接拼接：

```text
z = concat(h_a_aligned, h_b_aligned, h_e_aligned)
z: [B, 384]
```

然后进入两层 MLP：

```text
384 -> 256 -> 128 -> 输出
```

KAN head 的输入完全一样，也是 `[B, 384]`，只是把前两层 MLP 换成 KANLinear。  
实验中 KAN head 退化，说明在这种高维拼接隐藏空间里，KAN 不一定比 MLP 更合适。

可以这样回答老师：

> KAN head 是最直接的替换实验，但它没有显式告诉模型哪些维度属于药物 A、哪些属于药物 B、哪些属于细胞，也没有告诉模型两药应该先组成组合实体。因此它在 cold-drug 中退化比较明显。

### 6. 三元超图 HgKAN-HG 具体怎么做？

三元超图对应 `BatchHypergraphBuilder` 和 `KANHypergraphEncoder`。

在每个 batch 内，模型先收集唯一药物和唯一细胞系：

```text
节点类型1：drug节点
节点类型2：cell节点
```

如果 batch 中同一个药物出现多次，会把对应 embedding 平均，得到该 batch 内这个药物节点的特征。细胞系节点也一样。

然后每个样本构建一条三元超边：

```text
hyperedge_i = {drug_a_i, drug_b_i, cell_i}
```

超图的 `hyperedge_index` 可以理解为两行：

```text
第1行：节点索引
第2行：超边索引
```

例如第 i 个样本连接三个节点，就会加入三对关系：

```text
(drug_a_node, edge_i)
(drug_b_node, edge_i)
(cell_node, edge_i)
```

KAN 超图层是两阶段消息传递：

1. node -> hyperedge：把超边里的节点表示取平均，得到超边消息；
2. 对超边消息过 `edge_func`，如果是 HgKAN-HG，这里是 KANLinear；
3. hyperedge -> node：把与节点相关的超边消息再聚合回节点；
4. 对节点消息过 `node_func`，也是 KANLinear；
5. 加 skip connection。

公式化地说：

```text
e_k = mean(h_v, v 属于超边 k)
e_k' = KAN_edge(e_k)
m_v = mean(e_k', k 包含节点 v)
h_v' = skip(h_v) + KAN_node(m_v)
```

三层以后，再取回每个样本对应的 drug A 节点、drug B 节点和 cell 节点：

```text
h_a_enhanced = node_embeddings[drug_a_index]
h_b_enhanced = node_embeddings[drug_b_index]
h_e_enhanced = node_embeddings[cell_index]
```

最后再拼接三者进入 decoder。

这部分可以这样讲：

> HgKAN-HG 的好处是概念上很直接，因为一个药物协同样本天然就是 drug A、drug B、cell 三个实体共同决定的一条三元关系。缺点是药物节点和细胞节点类型不同，却在同一个超边空间里传播，KAN 同时要做实体类型对齐和非线性函数拟合，所以优化可能更困难。

### 7. HgKAN-Agg 药物对聚合图具体怎么做？

HgKAN-Agg 对应 `KANAggregatedGraphBuilder` 和 `KANAggregatedGraphEncoder`。这是本文 cold-drug 主结果最好的结构。

它不是直接保留 drug A、drug B、cell 三个节点，而是先把两个药物合成一个 drug-pair 节点：

```text
drug_pair_emb_i = (h_a_aligned_i + h_b_aligned_i) / 2
```

然后 batch 内节点变成两类：

```text
节点类型1：drug_pair节点
节点类型2：cell节点
```

每个样本至少建一条普通边：

```text
drug_pair_i <-> cell_i
```

此外，如果两个 drug-pair 共享某个药物，可以增加 pair-pair 边：

```text
drug_pair_i <-> drug_pair_j, 如果两者共享药物
```

这就是代码里的 `pair_edge_mode='shared_drug'`。也可以扩展成 molecular KNN 或随机 KNN，但主线可以讲 shared-drug。

然后用 `KANAggregatedGraphEncoder` 做三层图卷积。每层内部调用 `KAGCNConv`，也就是：

```text
neighbor_mean = mean(邻居节点表示)
combined = concat(当前节点表示, neighbor_mean)
updated = skip + KANLinear(combined)
```

输出之后，每个样本取回：

```text
drug_pair_emb = node_embeddings[drug_pair_index]
cell_emb = node_embeddings[cell_index]
```

因为 decoder 接口仍然需要三个输入 `h_a, h_b, h_e`，代码中把同一个 drug-pair embedding 复制给两个 drug 输入：

```text
h_a_enhanced = drug_pair_emb
h_b_enhanced = drug_pair_emb
h_e_enhanced = cell_emb
```

再拼接进入 decoder。

这部分建议你答辩时重点讲：

> HgKAN-Agg 的核心思想是先把两个药物组合成一个“药物对实体”，再让药物对和细胞背景交互。这样模型不需要在高维拼接向量里自己发现“哪两个药物应当组成组合”，而是结构上已经告诉它当前预测对象是 drug pair。KAN 只需要在 drug-pair 与 cell 的局部邻域里学习非线性消息函数，所以比直接 KAN head 或三元 KAN 超图更稳定。

### 8. decoder 最后怎么输出？

无论是 MLP、三元超图还是聚合图，最后都会得到三个增强后的表示：

```text
h_a_enhanced: [B, 128]
h_b_enhanced: [B, 128]
h_e_enhanced: [B, 128]
```

然后拼接：

```text
concat -> [B, 384]
```

进入 `HypergraphDecoder`：

```text
384 -> 256 -> 128 -> output
```

如果是分类：

```text
output: [B, 2]
LogSoftmax -> 非协同/协同概率
loss: NLLLoss
```

如果是回归：

```text
output: [B]
loss: MSELoss
```

训练中回归目标会做标准化：

```text
target_standardized = (target - train_mean) / train_std
```

评价时再还原到原始 Loewe 分数尺度，报告 RMSE 和 MAE。

## 三、几个实验怎么讲

### 实验一：Random split

这个实验回答“模型有没有基本拟合能力”。结果显示 MLP baseline 最强或接近最强，KAN head 直接替换没有收益。

答辩时可以这样说：

> Random split 是一个常规插值测试。它说明在已见药物空间内，普通 MLP 已经是很强的 baseline。KAN head 下降说明 KAN 不是通用替代品，尤其不适合直接处理高维拼接隐藏向量。

### 实验二：Cold-drug

这个实验回答“模型对未见药物有没有泛化趋势”。这是最重要的实验。

结果：

```text
MLP baseline: AUPR 0.2758, AUC 0.7333, RMSE 18.3997, MAE 13.6750
HgKAN-Agg:   AUPR 0.2765, AUC 0.7452, RMSE 17.6763, MAE 12.8615
```

答辩时注意语气：

> AUPR 提升很小，所以我不会把它说成显著领先。但分类 AUC、AUPR 和回归 RMSE、MAE 四个指标方向一致，说明 HgKAN-Agg 在当前实验设置下是最明确的正结果。

### 实验三：两 seed 稳定性

这个实验回答“结果是不是只靠一个 seed”。结论是：random 较稳定，cold-drug 波动明显；但聚合图方向比三元 KAN 超图更稳定。

可以这样讲：

> seed43 的 cold-drug 测试集更难，所以绝对指标下降明显。这也提示本文不能过度声称统计显著领先。更稳妥的结论是，药物对聚合图是更有潜力的结构方向。

### 实验四：Fourier 和 B-spline

这个实验回答“为什么用 Fourier KAN”。

答辩时可以说：

> B-spline 更适合有明确连续物理含义的变量，比如剂量。但本文 KAN 的输入是隐藏通道，不是真实剂量。实验中 B-spline 在 cold-drug 回归上退化，所以主实验采用 Fourier KAN。

### 实验五：参数量

这个实验回答“KAN 有没有更轻量”。

可以说：

> 没有。本文系统里细胞表达编码器参数很多，整体模型都在四千万参数级。KAN 会额外引入 0.9M 到 3M 多参数，因此本文不把参数效率作为贡献。

### 实验六：可解释性

这个实验回答“模型是否真的使用药物信息”。

答辩时可以说：

> 我用药物扰动和药物对表示恢复曲线检查模型依赖性。正样本中，屏蔽两药后预测概率从 0.9106 降到 0.3711；药物对表示恢复到 15% 时概率越过 0.5。说明模型输出确实对药物对结构敏感。原子 saliency 进一步把解释落到分子结构上，但这仍然是模型内部解释，不能替代湿实验机制验证。

## 四、老师可能追问时的简短回答

### Q1：为什么 HgKAN-Agg 比三元超图好？

三元超图直接把 drug A、drug B 和 cell 放在同一条超边里，概念上很自然，但药物节点和细胞节点是异质实体，KAN 需要同时做类型对齐和非线性函数学习。HgKAN-Agg 先把两个药物合并成 drug-pair 节点，再与 cell 交互，相当于先明确“预测对象是药物组合”，降低了 KAN 的学习负担，所以在 cold-drug 中更稳定。

### Q2：为什么 DrugKAN 分类不稳定，但回归有收益？

单药分子结构可以帮助预测总体效应强弱，所以对连续 Loewe 回归可能有帮助。但协同二分类衡量的是两药联合是否超过参考模型期望，它更依赖组合关系。单药表示增强不一定能改善组合协同判别。

### Q3：为什么 KAN head 失败？

因为 KAN head 处理的是 `[drug A embedding, drug B embedding, cell embedding]` 拼接后的高维隐藏向量。这些维度不是明确的药理变量，而是神经网络混合出来的表示。KAN 的单变量边函数需要相对稳定的输入语义，否则不一定比 MLP 更好。

### Q4：你的 KAN 曲线能解释药理机制吗？

不能直接解释为药理机制。KAN 曲线的横轴是隐藏通道激活值，不是剂量、靶点占有率或反应时间。它只能说明模型内部消息函数学到了非线性关系。真正的药理机制还需要实验验证。

### Q5：这个模型能直接用于临床联合用药推荐吗？

不能。本文模型适合做体外抗癌药物组合的候选排序，帮助缩小实验筛选范围。临床用药还涉及药代动力学、毒性、给药顺序、患者状态等因素，这些都没有纳入本文输入。

### Q6：本文最核心的一句话是什么？

KAN 在药物协同预测中不是 MLP 的通用替代品；它更适合作为结构化消息传递中的可学习非线性函数模块。

## 五、追问补充：drug 端到底用了哪个版本，batch 内边怎么加？

这一节专门回答老师如果追问“你 drug 端不是实现了很多版本吗，实验到底用哪个？”以及“超图预测时 batch 内的边从哪里来？”。

### 1. 先说结论：主实验用的是哪一个 drug 端？

我确实实现了很多 drug encoder 变体，包括：

1. 老版 `GATNet`；
2. 普通 `GCN/GAT/GATv2/GIN`；
3. KAN 版本的 `KAGCN/KAGAT/KAGIN`；
4. `KAGCNNeighborConv`；
5. `KAOriginalConv`；
6. `paper_kagcn`，也就是更接近 KA-GNN 论文公式的版本；
7. `concat_kan`、`multilayer JK`、`readout KAN` 等读出增强版本。

但是 PPT 和论文主表里的核心实验，为了控制变量，drug 端统一走 `KAGNN` 这个接口，具体配置是：

```text
gnn_type = gcn
num_layer = 5
emb_dim = 128
graph_pooling = mean
virtual_node = False
with_edge_attr = False
drug_jk = last
kan_type = fourier
expression_input_size = 946
```

差别只在 `use_drug_kan` 这个开关：

```text
use_drug_kan = False -> 普通 GCN drug encoder
use_drug_kan = True  -> DrugKAN，也就是 KAGCN drug encoder
```

所以答辩时不要说“主模型 drug 端用了所有版本”。更准确的说法是：

> 我实现了多个 drug encoder 变体，但论文主实验为了公平比较 KAN 放置位置，固定使用 5 层 GCN/KAGNN 框架。MLP baseline 和 HgKAN-Agg 主结果的 drug 端是普通 GCN；DrugKAN 和 DrugKAN+HgKAN-Agg 才是在 drug 端打开 KAN 的版本。

这里有一个容易混淆的点：`HgKAN-Agg` 这个名字指的是后面的药物对-细胞聚合图用了 KAN，不等于 drug 端也用了 KAN。表里 `DrugKAN+HgKAN-Agg` 才是 drug 端和后面交互端都用 KAN 的叠加版本。

### 2. drug 端输入是什么？

每个样本有两个药物：drug A 和 drug B。每个药物由 SMILES 解析成分子图：

```text
节点：原子
边：化学键
节点特征：离散原子属性
边索引：edge_index
边特征：edge_attr，当前主实验 drug 端没有显式使用
```

进入 batch 以后，PyTorch Geometric 会把多个分子图拼成一个大的不连通图。以 drug A 为例：

```text
batch.x_a          所有 drug A 分子图的原子特征拼接
batch.edge_index_a 所有 drug A 分子图的边索引拼接，并自动偏移节点编号
batch.edge_attr_a  所有 drug A 分子图的边特征
batch.x_a_batch    每个原子属于 batch 中第几个样本
```

drug B 同理：

```text
batch.x_b
batch.edge_index_b
batch.edge_attr_b
batch.x_b_batch
```

模型分别编码 drug A 和 drug B，两个药物共享同一个 `drug_encoder` 参数。

### 3. drug 端内部数据流

drug 端代码路径是：

```text
DeepDDS_Hypergraph._encode_modalities
-> _encode_drug
-> KAGNN.forward
-> KAGNN_node.forward
-> GCNConv 或 KAGCNConv
-> global_mean_pool
```

更具体地说：

第一步，原子离散特征进入 `AtomEncoder`：

```text
x: [batch内所有原子数, 原子特征维度]
-> h0: [batch内所有原子数, 128]
```

第二步，做 5 层图消息传递。  
如果 `use_drug_kan=False`，每一层是普通 `GCNConv(128,128)`。  
如果 `use_drug_kan=True`，每一层是 `KAGCNConv(128,128)`。

`KAGCNConv` 的核心更新是：

```text
neighbor_mean_i = mean(h_j, j 属于 i 的邻居)
combined_i = concat(h_i, neighbor_mean_i)
transformed_i = KANLinear(combined_i) 或 Linear(combined_i)
h_i' = skip(h_i) + transformed_i
```

也就是说，DrugKAN 不是把整个药物对直接丢给 KAN，而是在原子图的局部邻域里使用 KAN。当前原子表示和邻居平均表示先拼接，然后过一层 KANLinear，最后加残差。

第三步，每层后面接：

```text
BatchNorm
ELU/ReLU
Dropout
```

主实验用 `JK=last`，所以最后只取第 5 层节点表示。

第四步，图级池化：

```text
global_mean_pool(h_node, x_a_batch)
-> h_a_raw: [batch_size, 128]

global_mean_pool(h_node, x_b_batch)
-> h_b_raw: [batch_size, 128]
```

然后 cell 端表达编码器输出：

```text
expression: [batch_size, 946]
-> h_e_raw: [batch_size, 256]
```

最后经过 `EmbeddingAligner` 对齐维度：

```text
h_a_raw: [B,128] -> h_a_aligned: [B,128]
h_b_raw: [B,128] -> h_b_aligned: [B,128]
h_e_raw: [B,256] -> h_e_aligned: [B,128]
```

答辩时可以压缩成一句：

> drug 端先把 SMILES 转成分子图，再把 batch 内所有分子拼成不连通大图；原子特征经过 AtomEncoder 后做 5 层 GCN/KAGCN 消息传递，最后用 mean pooling 得到每个药物 128 维表示。

### 4. 三元超图 HgKAN-HG 的 batch 内边依据

三元超图不是从外部知识库查边，也不是用全数据集预先建一个大图。训练和测试时都是只根据当前 mini-batch 里的样本即时建图。

对于一个 batch，先收集当前 batch 出现过的唯一药物和唯一细胞系：

```text
drug nodes = unique(batch.drug_a_id 和 batch.drug_b_id)
cell nodes = unique(batch.cell_id)
```

如果同一个药物或同一个细胞系在 batch 里出现多次，就把它对应的 embedding 求平均，作为这个 batch 内的节点特征。

然后每一行样本加一条三元超边：

```text
hyperedge_i = {drug_a_i, drug_b_i, cell_i}
```

也就是说，边的依据就是原始数据中的一条药物组合实验记录：这条记录本身说明 drug A、drug B 和 cell 共同决定一个协同标签，所以它们被连成一条超边。

超图消息传递是：

```text
node -> hyperedge:
e_i = mean(h_drugA, h_drugB, h_cell)

hyperedge function:
e_i' = KAN_edge(e_i)

hyperedge -> node:
每个节点收集包含自己的超边消息并求平均

node function:
h_v' = skip(h_v) + KAN_node(message_v)
```

如果老师问“测试时会不会用了测试集以外的信息”，可以回答：

> 不会。代码里明确是当前 query batch 内建图，每行样本只加自己的 drug-drug-cell 超边，不检索额外 context 样本，也不把训练集样本拿来参与测试 batch 建图。

### 5. HgKAN-Agg 聚合图的 batch 内边依据

HgKAN-Agg 不是三元超图，而是普通图。它先把两个药物合成一个药物对节点：

```text
drug_pair_emb_i = (h_a_aligned_i + h_b_aligned_i) / 2
drug_pair_id_i = sorted(drug_a_id_i, drug_b_id_i)
```

这里 `sorted` 是为了让 `(A,B)` 和 `(B,A)` 被视为同一个药物对。

然后 batch 内节点有两类：

```text
drug_pair 节点
cell 节点
```

边有两种来源。

第一种是每个样本都一定会加的 pair-cell 边：

```text
drug_pair_i <-> cell_i
```

依据是这条样本本身就是“某个药物对在某个细胞系上的协同实验”。所以 drug-pair 和 cell 必须连接，而且代码里是双向边。

第二种是可选的 pair-pair 边，由 `pair_edge_mode` 控制。代码支持：

```text
none
shared_drug
molecular_knn
shared_plus_molecular_knn
random_knn
```

主实验命令默认是 `pair_edge_mode='shared_drug'`。这个模式的规则是：

```text
如果两个 drug_pair 共享任意一个药物，就在它们之间加双向边。
```

例如：

```text
(A, B) 和 (A, C) 共享 A -> 加边
(A, B) 和 (D, E) 没有共享药物 -> 不加边
```

所以 HgKAN-Agg 的图结构可以理解为：

```text
每条样本：drug_pair-cell 边
同一 batch 内共享药物的组合：drug_pair-drug_pair 边
```

随后用 3 层 `KAGCNConv` 做消息传递：

```text
neighbor_mean = mean(邻居节点表示)
combined = concat(当前节点表示, neighbor_mean)
updated = skip + KANLinear(combined)
```

最后对每个样本取回：

```text
drug_pair_emb = node_embeddings[drug_pair_index]
cell_emb = node_embeddings[cell_index]
```

因为 decoder 仍然接收三个输入，所以代码把同一个 drug-pair embedding 复制给 drug A 和 drug B：

```text
h_a_enhanced = drug_pair_emb
h_b_enhanced = drug_pair_emb
h_e_enhanced = cell_emb
```

再拼接预测：

```text
concat([h_a_enhanced, h_b_enhanced, h_e_enhanced])
-> [B,384]
-> decoder
-> 分类 log probability 或回归 Loewe score
```

### 6. 这段答辩怎么说最稳？

可以这样回答：

> 我实现过多种 drug encoder，但主实验为了控制变量，固定使用 5 层 GCN/KAGNN 框架。普通 baseline 的 drug 端是 5 层 GCN；DrugKAN 是把每层 GCN 的线性更新换成 KAGCNConv 里的 KANLinear。主结果 HgKAN-Agg 的重点不在 drug 端，而在后面的药物对-细胞交互图：先把两个药物平均成 drug-pair 节点，每条样本加 drug-pair 到 cell 的边；如果两个药物对共享一个药物，再加 drug-pair 到 drug-pair 的边。整个图只在当前 batch 内构建，测试时不引入训练集额外样本。
